codePush

Feb 20, 2017 | Hits

​ React Native是以js脚本语言来编写，并由客户端框架来解释并最终调用native环境来运行的一种新的移动端架构，通过以js脚本来编写，可以显著提高应用的开发效率，为移动端开发带来了跨平台、免编译调试等特性，同时由于解释执行的特点，通过替换js脚本可以使应用原生带有热更新的功能，比如React Native的开发模式下客户端reload js脚本其实就是热更新的应用。

​ 但Facebook并没有提供官方的React Native热更新API，要实现热更新功能需要比较深入得研究React Native的源码，同时部署远程升级服务器和升级版本管理等功能也是比较大的工作量，所以微软在github上推出了开源的CodePush热更新框架，可以大幅减少RN平台下热更新的工作量。

​ CodePush目前最新的版本是v1.17.0+，支持到RN最新的0.41版本，官方说会根据RN版本的更新同步更新。支持

  • iOS (7+)

  • Android (4.1+)

  • Windows (UWP)

这三个主流的移动平台，下面主要介绍CodePush在Android平台上的部署，基于RN 0.41。

CodePush比较特别的是所有控制都是通过命令行执行，而没有像其他网络服务一样通过网页来配置。

  • get key of your app

    运行这个命令可以获取到当前项目的deployment key，默认分为Production和Staging两个阶段，可以分别用于产品发布和测试阶段，应用程序通过对应的key去服务器下载对应的升级文件，当然也可以手动添加其他阶段项。
