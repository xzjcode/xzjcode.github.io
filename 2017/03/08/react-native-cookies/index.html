react-native Cookies note

Mar 8, 2017 | Hits

​ 在包含React Native模块的项目开发中，经常会用到在原生Native模块和RN模块间的登陆状态共享的需求。一般在应用中登陆后http服务器会返回cookies记录登陆状态，再交由系统保存和管理。在Android系统中，自带有 CookieManager 和CookieSyncManager(deprecated in API21)来管理cookies。

Android的cookies管理

​ 如果使用Android自带的WebView，Android系统会自动进行cookies的管理，因为从性能方面考虑，cookies会优先保存在RAM中，并由后台线程定期保存到存储模块进行持久化。持久化的时间间隔默认是5min。CookieSyncManager

The WebView now automatically syncs cookies as necessary. You no longer need to create or use the CookieSyncManager.

The CookieSyncManager is used to synchronize the browser cookie store between RAM and permanent storage. To get the best performance, browser cookies are saved in RAM. A separate thread saves the cookies between, driven by a timer.

The sync interval is 5 minutes, so you will want to force syncs manually anyway.

###React Native的cookies管理

​ React Native中一般使用fetch()进行http请求，通过查看RN 网络底层源码NetworkingModule.java，可以发现，fetch()在底层由OkHttp实现，cookies管理相关源码则在ForwardingCookieHandler.java中，ForwardingCookieHandler实现了抽象类CookieHandler的一系列cookies管理方法，并且在内部类CookieSaver中，实现了通过调用CookieManager 和CookieSyncManager实现了cookies的持久化，并且持久化的时间隔缩短为30s。由于底层的cookies管理还是通过Android默认的CookieManager 和CookieSyncManager，所以就实现了RN与Android系统cookies的共享。
