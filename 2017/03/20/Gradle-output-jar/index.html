Mar 20, 2017 | Hits

Android studio 打包 Jar

​ 对于Library类型，Android studio默认build后会打包成aar格式，aar同时包含了class文件和资源文件，比java标准的Jar格式更为方便，但是Eclipse是不支持导入aar的，对于SDK而言，Jar无疑兼容性会更高。所以需要通过自定义Gradle脚本的方式在Android Studio中导出Jar格式的库。

导出Jar库

​ 主要思路是在项目Gradle脚本中新建一个task，依赖于compileDebugJavaWithJavac或者compileReleaseJavaWithJavac，定义好name、version、from路径、include路径、exclude路径等参数即可。其中from的参数可以通过zipTree方法直接获取到第三方库jar包里的文件。主要代码：

Jar中打包图片资源

​ 由于Jar的限制，默认是无法打包图片的，可以采用一个变通的方法，就是将图片添加到项目的asserts目录下，打包的时候include进来，应用include这个Jar的时候，会将asserts合并到应用的asserts目录中，Jar中的Java代码可以通过AssetManager访问图片资源。

混淆Jar

​ 通过dependsOn之前的打包task并且制定type为proguard.gradle.ProGuardTask就可以将刚刚打包好的jar进行混淆，只需要配置好injars、outjars和configuration就可以了。

局限

​ 这种方式可以将图片资源文件打包进Jar，但是对于layout、value等xml还是没有找到比较好的方法，目前项目中只能采用纯java布局的方式绕过这方面的限制，后续会在这方面继续研究。
