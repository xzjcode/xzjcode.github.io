React-Native life cycle

Mar 28, 2017 | Hits

React—Native生命周期

1. 初始化（Initialization）

  • getDefaultProps

  • getInitalState

  • componentWillMount

    render前调用，在此设置state不会触发re-render，只调用一次

  • render

    返回需要包含DOM的RN single child component/ null / false(不需要render的情况)

    DOM相关操作，State的改变都不应该出现在这个函数中

  • componentDidMount

    render后调用，在此可以操作RN DOM，只调用一次，RN component的ref回调会晚于componentDidMount，不要使用依赖ref回调的方法来操作RN DOM，

####2. 状态改变（State Changes）

  • shouldComponentUpdate

    根据返回true/false来确定是否进行re-render，可以控制控件的更新策略，提高render性能，StaticContainer:

  • componentWillUpdate

    shouldComponentUpdate后调用，为render作准备，不应该在此更新state

  • render

  • componentDidUpdate

    render后调用

####3. 卸载控件（Unmounting）

  • componentWillUnmount

    从DOM卸载控件时调用，用来移除比如timer或listener

####4. some more

  • 页面跳转时的回调

    有时会需要back到上一个页面时进行一些操作，在Android原生程序中这部分是在生命周期中处理，但是在RN中这块回调是在Navigator中的，需要在Navigator的onDidFocus属性中注册回调函数，这部分和原生应用的习惯不一样。这个方法仅限于RN默认的Navigator组件，官方现在推荐使用性能更好的React Navigation，回调方法应该会有变化。

  • 应用状态改变回调

    类似于Android原生应用的onSaveInstanceState，RN应用也需要处理被调度到后台后可能会被系统回收的情况，可以通过AppState来注册回调
